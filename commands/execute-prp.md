# Exécuter un PRP

## Fichier PRP : $ARGUMENTS

Implémente la feature décrite dans le PRP.

## Processus d'exécution

1. **Charger le PRP**
   - Lire le fichier PRP complet
   - Comprendre tout le contexte et les exigences
   - Suivre toutes les instructions du PRP
   - Approfondir la recherche si nécessaire (web, codebase, vault MCP)

2. **ULTRATHINK**
   - Réfléchir en profondeur avant d'exécuter
   - Créer un plan détaillé couvrant TOUTES les exigences
   - Découper les tâches complexes en étapes gérables
   - Identifier les patterns existants à suivre dans les skills

3. **Exécuter le plan**
   - Implémenter chaque tâche dans l'ordre défini par le PRP
   - Respecter les conventions : encodage, nommage, frontmatter
   - Utiliser les wikilinks [[Nom]] dans les notes vault
   - Ne jamais modifier les _Templates/ sauf demande explicite

4. **Valider**
   - Exécuter chaque commande de validation du PRP
   - Niveau 1 : Structure (fichiers existent, encodage correct)
   - Niveau 2 : Fonctionnel (commandes marchent, scripts s'exécutent)
   - Niveau 3 : Intégration (router détecte, vault indexé)
   - Corriger les échecs et re-tester jusqu'à ce que tout passe

5. **Compléter**
   - Vérifier que tous les items de la checklist sont faits
   - Exécuter la suite de validation finale
   - RELIRE LE PRP pour s'assurer que tout est implémenté
   - Rapporter le statut de complétion

6. **Référence**
   - Le PRP peut être relu à tout moment si besoin

Note : Si la validation échoue, utiliser les patterns d'erreur du PRP pour corriger et réessayer.
