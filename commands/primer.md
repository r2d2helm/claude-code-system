# Primer - Analyse rapide d'un projet

## Cible : $ARGUMENTS

Analyse le projet ou dossier spécifié pour en comprendre la structure et le fonctionnement.

## Processus

1. **Structure**
   - Lister l'arborescence du projet (tree ou ls récursif)
   - Identifier les fichiers clés (config, entry points, README)

2. **Documentation**
   - Lire CLAUDE.md si présent
   - Lire README.md si présent
   - Lire les fichiers de configuration (package.json, pyproject.toml, etc.)

3. **Code source**
   - Lire les fichiers principaux (main, index, app)
   - Identifier les dépendances et leur rôle
   - Comprendre l'architecture (patterns, structure)

4. **Rapport**

Expliquer clairement :
- **Structure** : organisation des dossiers et fichiers
- **Objectif** : but du projet et cas d'usage
- **Fichiers clés** : rôle de chaque fichier important
- **Dépendances** : librairies et services externes
- **Configuration** : variables d'environnement, fichiers de config
- **Patterns** : conventions de code, architecture utilisée
- **Points d'attention** : gotchas, limitations, dette technique
